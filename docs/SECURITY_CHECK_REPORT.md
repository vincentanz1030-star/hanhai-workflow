## 一、已修复问题汇总

### 1. 权限保存问题 ✅ 已修复
**问题描述**: 编辑权限时，保存提示"无效的请求参数"

**根因分析**: 
- 前端发送 `{ grant: [...], revoke: [] }` 格式
- 后端API期望 `{ type: 'permissions', permissions: [...] }` 格式

**解决方案**: 更新 `/api/admin/users-v2/[id]/permissions` API，支持多种请求格式：
- 格式1: `{ type: 'permissions', permissions: [...] }`
- 格式2: `{ grant: [...], revoke: [...] }`
- 格式3: `{ permission_ids: [...] }`

### 2. 备份创建后无记录 ✅ 已修复
**问题描述**: 点击创建备份成功后，备份列表无记录

**根因分析**:
- API插入字段与数据库表结构不匹配
- API使用: `file_size, record_count, tables, created_at`
- 数据库实际: `size, type, backup_date`

**解决方案**: 更新 `/api/backups` API，使用正确的表字段：
```typescript
{
  name,
  type: 'full',
  size: JSON.stringify(backupData).length,
  file_path: null,
  backup_date: new Date().toISOString(),
  created_by: authResult.userId,
  is_deleted: false,
}
```
