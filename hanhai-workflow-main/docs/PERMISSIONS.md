# 权限管理文档

## 概述

本文档描述瀚海集团工作流程管理系统的权限管理体系，包括角色定义、权限配置和管理方法。

## 权限体系架构

### 数据库表结构

系统使用三个表来管理权限：

1. **permissions** - 权限定义表
   - id: 权限ID (UUID)
   - resource: 资源类型 (如 project, task, product)
   - action: 操作类型 (如 view, create, edit, delete)
   - description: 权限描述

2. **role_permissions** - 角色权限映射表
   - role: 角色名称
   - permission_id: 权限ID

3. **user_roles** - 用户角色表
   - user_id: 用户ID
   - role: 角色名称
   - is_primary: 是否为主角色

## 权限使用方法

### 在 API 中检查权限

```typescript
import { requireAuth } from '@/lib/api-auth';

export async function GET(request: NextRequest) {
  // 检查用户是否有 project:view 权限
  const authResult = await requireAuth(request, 'project', 'view');
  if (authResult instanceof NextResponse) {
    return authResult; // 返回 401 或 403 错误
  }

  // 继续处理请求
  const { userId, email, brand, roles } = authResult;
  // ...
}
```

### 在客户端检查权限

```typescript
import { checkPermission } from '@/lib/permissions';

const hasPermission = await checkPermission(userId, 'product', 'create');
if (!hasPermission) {
  // 显示无权限提示
}
```

## 权限管理最佳实践

### 1. 最小权限原则
只授予用户完成任务所需的最小权限，避免过度授权。

### 2. 角色分离
不同岗位使用不同角色，避免一个用户拥有过多角色。

### 3. 定期审查
定期审查用户权限，及时收回不必要的权限。

### 4. 审计日志
记录所有权限变更和重要操作，便于追溯。

### 5. 敏感操作二次确认
对于删除、审批等敏感操作，增加二次确认。

## 常见问题

### Q: 如何添加新角色？
A: 在 `src/app/api/admin/permissions/init/route.ts` 中的 `ROLE_PERMISSIONS` 数组中添加新角色配置。

### Q: 如何添加新权限？
A: 在 `PERMISSIONS` 数组中添加新的权限定义，然后在相关角色配置中引用。

### Q: 如何修改角色的权限？
A: 修改 `ROLE_PERMISSIONS` 数组中对应角色的 `permissions` 字段，然后调用初始化 API。

### Q: 权限检查失败会怎样？
A: API 会返回 403 Forbidden 错误，提示用户无权限执行此操作。

### Q: 用户可以有多个角色吗？
A: 是的，一个用户可以拥有多个角色。系统会检查用户所有角色的权限并集。
